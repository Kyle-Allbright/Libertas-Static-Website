STREAMS HAVE UNIQUE CONTENT IDENTIFIERS

Content identifiers are used to point to files on the blockchain since they are not statically hosted. It does not indicate where the content is stored exactly but acts an address based on the underlying content.



DISTRIBUTED IN CHUNKS

Stream segments are split into chunks with unique IDs so they can be found again on the blockchain.





Tradional methods of playing audio streams host the media file on a centralized server in one whole chunk. The screenshot above displays the segmented podcast output by the Libertas MP3 conversion script that was then unploaded to the blockchain.

By storing the streams as segments we are able to use pointers to quickly access our content from anywhere on the network making retrieval fast. The above podcast is censorship resistant and the CID for it can be ran on a local node by downloading the desktop client: ipfs.io

At Libertas, we are creating a way for everyone to be able to upload content. Behind the scenes our technology will ensure you, the user, are given a seamless user interface without all of these technical aspects cluttering your experience.
